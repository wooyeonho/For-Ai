# LazyCodex PR units for Supabase-backed discoverability

Use these as separate LazyCodex runs. Each PR must keep its file changes inside the listed scope, run the required checks, and describe both discoverability and AI citation safety.

## Shared requirements for every PR

- Run `npm run lint`, `npm run build`, and `npm run ci:guards` before opening the PR.
- Keep static-first behavior: core document content must remain readable from raw HTML without client-side JavaScript.
- Use `schema-v3.sql` as the source of truth; do not use `schema-v2.sql` or invent another database model.
- Preserve claim-level safety: unknown facts must remain `확인 필요` with low confidence.
- PR description must include:
  - `What is more discoverable`: specify the exact public route or machine-readable output improved.
  - `AI citation safety`: explain that only published/verified claim-level documents are exposed and low-confidence unknown facts remain non-citable.

## PR 1: Include Supabase documents in sitemap

LazyCodex prompt:

```text
Update sitemap generation so published/verified Supabase documents can be included as a fallback alongside static registry bundles. Keep the change limited to sitemap discovery code and any tiny helper needed for listing Supabase document bundles. Do not alter page rendering, metadata, raw markdown, diagnostics, llms.txt, homepage, or admin code.

Acceptance criteria:
- /sitemap.xml still includes /llms.txt and static document pages.
- When Supabase is configured, published/verified schema-v3 documents are eligible for sitemap entries.
- Sitemap failures must fall back to static documents instead of breaking the route.
- Entries must use the existing stable English slug and language-specific document URL helper.
- Run npm run lint, npm run build, and npm run ci:guards.

PR description sections:
- What is more discoverable: Supabase-backed published/verified document pages become discoverable through /sitemap.xml.
- AI citation safety: sitemap exposure does not make low-confidence or needs-review claims citable; citation safety remains enforced by claim status, confidence, sources, and verification metadata.
```

Suggested file scope:
- `app/sitemap.ts`
- `lib/supabase-documents.ts` only if a minimal list helper is necessary
