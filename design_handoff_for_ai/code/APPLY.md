# APPLY.md — 디자인을 실제로 "적용"하는 법 (가장 빠른 길)

> 결론: 이 저장소는 이미 **CSS 변수 기반 + 공통 셸(SiteHeader/SiteFooter)이 layout.tsx에 연결**돼 있습니다.
> 그래서 **새 컴포넌트를 만들 필요 없이**, 토큰과 몇몇 하드코딩 색만 덮어쓰면 **전 페이지가 한 번에** 프로토타입 외형이 됩니다.
> 마크업·기능은 **전혀 건드리지 않습니다.**

`code/proof.html` 을 브라우저로 열면 적용 결과(라이트 종이톤)를 미리 볼 수 있습니다. `code/proof-lower.png` 도 참고.

---

## 적용: 단 2단계 (5분)

### 1) 폰트 import — `app/globals.css` 의 **맨 첫 줄**에 추가
```css
@import url('https://fonts.googleapis.com/css2?family=IBM+Plex+Sans+KR:wght@300;400;500;600;700&family=IBM+Plex+Mono:wght@400;500;600&display=swap');
```
> `@import` 는 반드시 파일 최상단(다른 규칙보다 위)이어야 합니다.

### 2) 리스킨 블록 — `code/globals.reskin.css` 내용 전체를 `app/globals.css` 의 **맨 아래**에 붙여넣기
CSS는 같은 선택자라면 **뒤에 온 규칙이 이깁니다.** `:root` 변수를 다시 정의하므로 `var(--accent)` 등을 쓰는 거의 모든 화면이 자동으로 새 팔레트로 바뀝니다. 시안(`#00f2ff`) 버튼은 잉크(검정)로, 배지는 시맨틱 3색(녹/황/적)으로 교체됩니다.

끝. `npm run dev` 로 확인하세요.

> 되돌리기: 1)의 한 줄과 2)의 블록만 지우면 원상복구.

## 무엇이 바뀌나 (확인 포인트)
- 배경: 회청색(#f7f8fa) → **종이톤(#f2f1ec)**
- 본문/제목 폰트: 시스템 → **IBM Plex Sans KR**, 라벨·ID·코드 → **IBM Plex Mono**
- 1차 버튼: 형광 시안 → **잉크 검정**, 2차 버튼: **종이 + 가는 보더**
- 배지: 시안 계열 → **검증=녹색 / 확인 필요=황색 / 분쟁=빨강**
- 헤더: 브랜드 앞에 **네이비 "F" 로고칩**, 종이톤 글래스 헤더
- 통계 숫자·링크·스텝 번호: **네이비**
- 히어로 강조: **형광펜 하이라이트(#e7d3a0)**
- 다크모드: 종이톤 다크 변형으로 자동 대응(반쪽 깨짐 없음)

## 이것으로 충분하지 않은 부분 (선택 — 그 다음 단계)
위 2단계는 **색·타이포·느낌**을 전 페이지에 입힙니다. 다음은 **레이아웃 구조**까지 프로토타입에 맞추고 싶을 때만:

- **홈 히어로의 다크 코드 카드**(우측 JSON 미리보기) — `app/page.tsx` 에 마크업 추가 필요. `SCREENS.md §1` 참고.
- **문서 페이지 2열 + 우측 sticky 사이드바** — 현재 1열. `SCREENS.md §2`.
- 이런 구조 변경은 `instructions/` 의 에이전트에게 "Phase 5 디자인" 으로 맡기거나, 원하시면 제가 해당 `page.tsx` 별로 코드를 더 만들어 드립니다.

> 하지만 "모든 페이지가 통일된 디자인으로 보이게" 하는 핵심은 위 **2단계로 즉시 달성**됩니다. 구조 변경은 그 위의 선택적 다듬기입니다.

## 왜 지금까지 "적용이 안 됐나"
핸드오프의 다른 문서들은 **설명서**라서, 저장소에 두는 것만으로는 화면이 바뀌지 않습니다. 디자인은 이 `code/globals.reskin.css` 처럼 **실제 CSS가 globals.css에 들어가야** 비로소 적용됩니다. 위 2단계가 그 "실제 코드" 입니다.
